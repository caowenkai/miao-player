# 快速开始指南

## 10 分钟上手喵音乐播放器 🎵

### 第一步：克隆或下载项目

如果你已经有项目代码，跳过此步骤。

```bash
git clone <repository-url>
cd miao-player
```

### 第二步：一键部署（开发环境）

#### 方式一：使用部署脚本（推荐）

```bash
bash deploy.sh
```

脚本会自动：
- 安装所有依赖
- 创建必要目录
- 配置环境变量
- 启动后端服务

#### 方式二：手动安装

```bash
# 1. 安装后端依赖
cd backend
npm install

# 2. 安装前端应用端依赖
cd ../frontend-app
npm install

# 3. 安装后台管理依赖
cd ../frontend-admin
npm install
```

### 第三步：启动开发服务

#### 使用快速启动脚本（推荐）

```bash
bash start-dev.sh
```

这会同时启动所有三个服务。

#### 或者分别启动

**终端 1 - 启动后端：**
```bash
cd backend
npm run dev
```

**终端 2 - 启动前端应用端：**
```bash
cd frontend-app
npm run dev
```

**终端 3 - 启动后台管理：**
```bash
cd frontend-admin
npm run dev
```

### 第四步：访问应用

- 🎵 **前端应用端**: http://localhost:5173
- 🔧 **后台管理**: http://localhost:5174
- 📡 **后端 API**: http://localhost:3000

### 第五步：开始使用

1. **访问后台管理系统** (http://localhost:5174)
2. **点击"音乐管理" → "上传音乐"**
3. **上传你的音乐文件** (支持 mp3, wav, flac, m4a, ogg)
4. **点击"歌单管理" → "创建歌单"**
5. **创建一个歌单，并添加刚上传的音乐**
6. **访问前端应用端** (http://localhost:5173)
7. **点击底部"歌单"标签，选择你创建的歌单**
8. **点击歌曲开始播放！** 🎉

## 常见问题

### Q1: 端口被占用怎么办？

修改配置文件：
- 后端端口：编辑 `backend/.env`，修改 `PORT=3000`
- 前端应用端：编辑 `frontend-app/vite.config.js`，修改 `server.port`
- 后台管理：编辑 `frontend-admin/vite.config.js`，修改 `server.port`

### Q2: 上传音乐失败？

确保：
1. `uploads` 目录存在且有写入权限
2. 音频文件格式正确
3. 文件大小合理（建议不超过 50MB）

### Q3: 无法播放音乐？

检查：
1. 浏览器控制台是否有错误
2. 后端服务是否正常运行
3. 音频文件路径是否正确

### Q4: 如何重置数据库？

```bash
# 删除数据库文件
rm backend/data/database.db

# 重启后端服务，数据库会自动重建
cd backend
npm run dev
```
